Getting started

This tutorial should take 20 minutes. It will cover the basics of using the tool using the example tiles that come with the asset, so you won't need anything more than Unity and a license for the tool.

We will import a tileset and start editing a level using it.

<gif of editing a level>

Installing Voxel Collapse

Purchasing Voxel Collapse

If you haven't already, you will need to buy the Voxel Collapse asset from the Unity Asset Store.

Downloading and Installing Voxel Collapse into your Project

Voxel Collapse uses the standard flow for installing packages from the Asset Store.

Once bought, it will show up in the Unity Editor Package Manager. In Unity, select Window→Package Manager.

<gif of selecting package manager>

In the package manager, click on the Packages drop down button and select My Assets to view the assets you have purchased. If you cannot see Voxel Collapse in the list on the left, you can search for it in the top right.

Once you have found and clicked on Voxel Collapse, click on Download in the bottom right. After downloading, this button will change to Import. Click it again to import the package into your project. Import the whole project including the examples folders, we will be using models from the examples later in this tutorial.

<gif of selecting My Assets and Voxel Collapse, then downloading it>

Setting up the Tileset

Creating the Importer and the Tileset assets

The importer stores information about how Voxel Collapse should import your models so that you can edit the model externally and reimport it reliably. The importer asset typically does not need to be delivered with your built game (Unity will automatically ignore it as long as nothing is referencing it).

The tileset asset stores the processed information about how the models fit together. Unity will include the tileset asset when building your game if you use the GridPlacer component.

In the Project window, navigate to Assets and right click anywhere in the right window and select Create→Folder. Name this folder Voxels, then double click on the new folder to open it.

<gif of creating the folder and opening it>

Right click on the right window again and select Create→Voxel Collapse→Model Module Importer to create the importer. Right click again and select Create→Voxel Collapse→Tileset to create the tileset.

Note

Since the tileset is included in the build and the importer is not, you may wish to separate them when you use them in your actual project.

Linking the assets

Select the importer asset in the project window to view the settings in the inspector window. Drag the tileset asset from the project window to the Destination Tileset field in the inspector. This tells the importer to store its output in the tileset asset when it processes the models.

In the inspector window click the target icon at the right of Source Model to open a selection window. In this window, find and click the KenneyLandscape asset. The importer constructs tiles from the submodels in this model asset.

<gif of selecting the importer asset and setting the destination tileset and source model>

Setting these two fields enabled one-click reimporting. Click on the Process models button to start the import process.

Note

The import process will take second or two (depending on your computer). As your tilesets grow you may need to wait longer when reimporting.

Creating a map asset

Finally, we will need a voxel grid to use as a map. Create a voxel grid asset by right clicking in the project window and selecting Create→Voxel Collapse→Voxel Grid.

Note

The map is a distinct asset, separate from the scene. This allows you to share maps between different scenes and grants you granular access in source control, helping more developers to work together on the same project.

Select the new voxel grid asset and change the dimensions to (10, 3, 10).

Note

This is the size of the voxel grid, when using the GridPlacer component the generated model grid will be 1 unit greater in every dimension. In this case the placer will generate models in a 11×4×11 grid.

Caution

Resizing the grid to smaller dimensions will lose any voxels that have been stored in the area outside the new dimensions.

<gif of creating the asset and setting the dimensions>

Setting up the Scene

Creating the Tilemap Game Object

Now that the tileset and the map are ready, you can start using them in your scene.

Create an empty game object in your scene by right clicking in the hierarchy window and selecting Create Empty. The new object should be automatically selected when you create it - make sure you can see it in the properties window.

<gif of creating the object>

In that properties window, click the Add Component button and select Grid Placer (you can type "Grid Placer" into the search bar to quickly find it). This will add a blank GridPlacer component to your game object.

In the grid placer properties, find the Tileset and Map properties. From the project window, drag your voxel grid asset to the Map property and your tileset asset to the Tileset property.

<gif of adding the component and populating it>

Nothing will happen yet, even if you click Force map rebuild, because the map starts empty.

Editing the Map

Make sure the object with the GridPlacer component is selected. Press U to start the voxel editing tool. Press W to return to normal view.

Note

If, at any time, you wish to exit the voxel editor tool, press W. This will select Unity's built in translate tool, shutting down the voxel editor tool. Pressing W will never affect your scene or the voxel grid.

You can also select any other tool or deselect the current object in the hierarchy window to turn off the voxel editing tool. These are 100% safe ways of exiting the tool.

Press U again to enter the voxel editing tool. This will show the inside of a cuboid grid. Click on grid squares to build outward. You can then click on the new voxels to build out from them. Control-clicking on a voxel will destroy the voxel.

Once you have created a voxel map you are happy with, press W to exit the editor tool and immediately generate a mesh that glues.

You can continue to iteratively build the map by entering the tool, making some changes, and exiting again to see how it has changed the output model.
