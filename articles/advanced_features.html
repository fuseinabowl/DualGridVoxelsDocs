Extending Wildtile

Wildtile provides support for injecting functionality using the Tileset postprocessing fields, which does not require any modification of Wildtile source code. This requires some simple scripting and supports a limited set of features.

Wildtile also provides the complete code in the asset store package - you are free to modify it as you like. This requires scripting experience and some time studying the code but grants you the ability to change Wildtile to match your game/app's requirements exactly.

Join the Wildtile Discord to get dev advice and support for extending Wildtile.

Map Postprocessors

Wildtile creates a grid of tile references in memory when creating the world. Postprocessors then turn the tile references into game objects.

Wildtile includes two postprocessors - the placer postprocessor and the welder postprocessor (default). Select the tileset importer and assign a new postprocessor creator. This will change how the postprocessing will work for that tileset.

Create a new postprocessor

You can also write your own postprocessors, which allows you to warp tiles, place props, or otherwise augment the tiles while placing them.

A postprocessing solution consists of a PostprocessorCreator and a Postprocessor. A PostprocessorCreator can be referenced from many TilesetImporter assets, whereas a Postprocessor is a backend class that the PostprocessorCreator will create during import and saved as a subasset of the Tileset. The designer typically will not make or edit any Postprocessors directly.
